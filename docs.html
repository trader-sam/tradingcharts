Quick start

import { createChart } from 'tradingcharts';

const chart = createChart(host);
chart.setData([{ time, open, high, low, close, volume }]);

Data lifecycle

// Adds a newer bar, replaces a matching timestamp, or inserts an older bar.
chart.update({ time, open, high, low, close, volume });

// Batched updates redraw once and normalize duplicate timestamps.
chart.updateMany(bars);

setData replaces the full dataset. update replaces a matching timestamp or appends a newer one; older timestamps are inserted in order. updateMany is the preferred path for feed bursts because it coalesces rendering. When viewing history, incoming bars preserve the viewport; at the live edge, the chart follows the latest bar.

All chart mutators return the chart for chaining. Bars, depth inputs, and series points must use finite numeric values; invalid public inputs throw a RangeError. TradingCharts is browser-only, so initialize it from client-side code after its host exists.

Plot your own data

chart.addSeries({
  id: 'my-signal', type: 'line', color: '#fbbf24', scale: 'price'
});
chart.setSeriesData('my-signal', [
  { time, value: 281.4 }, { time: nextTime, value: 283.1 }
]);

chart.addSeries({ id: 'score', type: 'histogram', scale: 'independent' });
chart.updateSeries('score', { time, value: 0.72 });

For an object-oriented workflow, use createSeries(options). It returns a handle with setData, update, and remove; inspect configured ids with getSeriesIds() and getPaneIds().

Series are user-owned data. Use line, histogram, area, or step-area; share the OHLC price scale or select an independent normalized scale, then remove them with removeSeries(id). Calculate EMA, MACD, order flow, or any other transform in your application.

Series-only domains

const chart = createChart(host, {
  primarySeriesVisible: false, ohlcTooltip: false
}).setXAxis({ type: 'numeric' });

chart.addSeries({
  id: 'curve', type: 'area', fillColor: '#1f5c72', negativeFillColor: '#743442'
});
chart.setSeriesData('curve', [
  { time: 0, x: -1, value: 0.12 },
  { time: 1, x:  0, value: 0.40 },
  { time: 2, x:  1, value: 0.12 }
]);

Custom series establish their own internal navigation domain when no OHLC bars are supplied. time remains the stable point identity; optional x provides numeric or ordinal axis values. No placeholder bars are required.

Custom lower panes

chart.addPane({ id: 'momentum', title: 'Momentum', height: 0.24 });
chart.addSeries({ id: 'rsi', pane: 'momentum', color: '#a78bfa' });
chart.setSeriesData('rsi', rsiPoints);

chart.removePane('momentum'); // also removes its assigned series

Named panes share the chart's time navigation, stack below price, and auto-scale from their own visible values. Drag inside a pane to pan its Y scale, drag its right axis to zoom Y, and drag its divider to resize. height is a plot-height fraction between 0.12 and 0.5.

Timeframes and X axes

// Any millisecond interval works: seconds, minutes, hours, days, or custom.
chart.setData(intradayBars);

// Use labels from a numeric or ordinal domain instead of calendar time.
chart.setXAxis({ type: 'ordinal', formatter: (value) => `Contract ${value}` });
chart.setData([{ time: 0, x: 'A', open, high, low, close }]);

Time axes use stable calendar anchors: January renders as the year, month starts remain fixed as zoom increases, then days, hours, minutes, and seconds are added between them.

Order-book depth

import { createDepthData } from 'tradingcharts';

const depth = createDepthData(bids, asks, midPrice);
chart.setData(depth.bars).setXAxis({
  type: 'numeric', formatter: (price) => `$${Number(price).toFixed(2)}`
});
chart.addSeries({ id: 'bids', type: 'step-area', color: '#7df27a' });
chart.addSeries({ id: 'asks', type: 'step-area', color: '#ff744d' });
chart.setSeriesData('bids', depth.bids);
chart.setSeriesData('asks', depth.asks);
chart.setOverlays([{ type: 'vertical-line', time: depth.midTime,
  label: String(depth.midPrice), detail: 'Mid market price' }]);

createDepthData sorts levels, accumulates each side from the mid price, and returns a numeric X domain plus bid/ask point collections. Use fillColor for explicit area fills, or negativeFillColor to color the portion below zero, and yAxis: 'both' when quantity labels should appear on each edge.

A vertical overlay’s label is rendered as a badge on the X axis at its anchor; its optional detail is a horizontal callout beside the middle of the line.

Live order-book ladder

import { createOrderBook } from 'tradingcharts';

const book = createOrderBook(host, {
  levels: 10,
  quoteCurrency: 'USDC',
  priceFormatter: (price) => price.toFixed(1)
});

book.setSnapshot({ bids, asks });
// Feed deltas at any cadence; updates are coalesced per animation frame.
book.update({ bids: [{ price: 61982.5, size: 250_000 }] });
// A zero size removes a price level.
book.update({ asks: [{ price: 61983, size: 0 }] });

createOrderBook preallocates its ladder rows and applies snapshots or price-level deltas without rebuilding the DOM. It is intended for fast live feeds; call destroy() when its host is permanently removed. See the live order-book example.

3D options surface

import { createOptionsSurface } from 'tradingcharts';

const surface = createOptionsSurface(host, {
  strikeLabel: 'MONEYNESS (S / K)',
  expiryLabel: 'TIME TO MATURITY',
  valueLabel: 'IMPLIED VOLATILITY',
  valueFormatter: (iv) => `${(iv * 100).toFixed(1)}% IV`
});

surface.setData([
  { strike: 95, expiry: 30, value: 0.29 },
  { strike: 100, expiry: 30, value: 0.23 },
  { strike: 105, expiry: 30, value: 0.27 },
  // Complete the strike × expiry grid for a continuous surface.
]);

createOptionsSurface is a standalone Canvas component for option-chain metrics such as implied volatility, skew, or model price. It renders a strike × expiry grid as a perspective surface, supports custom formatters and axis labels, two-axis drag orbiting, wheel zoom, and point hover inspection. See it running in the examples gallery.

Options chain and multi-leg selection

import { createOptionsChain } from 'tradingcharts';

const chain = createOptionsChain(host, {
  spot: 102.4,
  expirations: [{
    id: '2026-03-20', label: '20 Mar · 42d',
    rows: [{
      strike: 100,
      call: { bid: 4.10, ask: 4.25, impliedVolatility: 0.24, volume: 812, openInterest: 6_420 },
      put: { bid: 1.80, ask: 1.95, impliedVolatility: 0.23, volume: 526, openInterest: 5_180 }
    }]
  }],
  maxLegs: 6,
  onLegsChange: (legs) => saveDraftStrategy(legs)
});

chain.setSpot(103.15);
const legs = chain.getLegs();
chain.clearLegs();

createOptionsChain is a DOM component for fast contract selection. Quotes can include implied volatility, volume, and open interest. It provides expiration switching, a strike filter, call/put visibility, and an accessible multi-leg basket. Clicking a bid adds a sell leg; clicking an ask adds a buy leg. Expiration changes preserve selected legs so callers can compose calendars and diagonals. See the live strategy-builder example.

Streaming candles

// A new one-second candle.
chart.update({ time: previous.time + 1_000, open, high, low, close });

// Repeatedly replace the same active three-second candle as ticks arrive.
chart.update({ time: activeCandleTime, open, high, low, close });

update() replaces a matching timestamp and appends a newer one. This supports both fixed-interval bars and high-frequency intrabar updates without replacing the full dataset. See both patterns running in the live examples gallery.

Built-in conveniences

chart.setVolumeVisible(true);
chart.setEvents(events);

Volume and corporate events are optional layers for the market series.

Corporate events

chart.setEvents([
  { time, type: 'dividend', marker: {
      color: '#6ee7b7', icon: '$', shape: 'diamond', filled: false
  }}
]);

Markers support per-event color, icon, circle/square/diamond shape, and filled or outlined rendering. Clicking a marker opens its optional popup content directly in the chart.

Buy and sell arrows

chart.setTradeMarkers([
  { time: entryBar.time, side: 'buy', price: 281.42, label: 'Entry' },
  { time: exitBar.time, side: 'sell', label: 'Exit', color: '#ff8b9c' }
]);

setTradeMarkers renders caller-owned buy and sell arrows. When price is omitted, buy arrows sit below the bar and sell arrows sit above it; supply an execution price to anchor an arrow exactly. Replacing the marker array is efficient for a live execution feed. The one-second streaming example in the gallery adds an arrow with each new candle.

Drawings

chart.setDrawingTool('trendline');
chart.setDrawingTool('horizontal-line');
chart.setDrawingTool('free-draw');
const saved = chart.getDrawings();
chart.setDrawings(saved); // restore persisted drawings after panes are created
chart.clearDrawings();

Tools work in both price and user-created lower panes. Use getDrawings() to obtain a safe persistence snapshot, then setDrawings() to restore it. Create any referenced lower panes before restoring their drawings.

Options

const chart = createChart(host, {
  background: '#0b1020', textColor: '#cbd6ee',
  upColor: '#38d39f', downColor: '#ff6b81', crosshair: true,
  ohlcTooltip: 'fixed',
  ohlcLegend: { title: 'ACME · 1 day', showVolume: true },
  padding: { right: 68, bottom: 32 }
});

Wheel over the plot zooms around the cursor. Wheel or drag on a price axis scales that pane; dragging a lower-pane divider resizes it. The crosshair exposes both a price badge and an X-axis badge. Set ohlcTooltip: 'fixed' for a compact, non-floating OHLC legend that stays at the top-left of the plot; it still updates as the crosshair moves.

Hover the fixed Vol. legend or a removable main-pane series to reveal its compact floating control. The control can hide volume or remove the series; give custom series a label and set removable: false when that action should not be offered.

Custom-series-only charts

const chart = createChart(host, {
  primarySeriesVisible: false,
  ohlcTooltip: false
});
chart.addSeries({ id: 'distribution', type: 'area', color: '#7dd3fc' });
chart.setSeriesData('distribution', points);

Use these options for charts powered solely by custom data. The distribution demo uses this exact public API—there are no demo-only rendering paths.

Scatter plots

const samples = chart.createSeries({
  id: 'samples', type: 'scatter', color: '#7dd3fc', markerRadius: 4
});
samples.setData(observations.map((sample, index) => ({
  time: index, x: sample.feature, value: sample.target
})));

chart.createSeries({ id: 'fit', type: 'line', color: '#ff8c9b' })
  .setData(regressionLine);

Scatter is a first-class custom series. It uses the same numeric, ordinal, or time X domain as every other series, so fitted lines, confidence bands, centroids, and clusters are simply additional caller-owned series. See the interactive scatter example.
