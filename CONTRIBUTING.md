# Contributing to TradingCharts

Thanks for helping improve TradingCharts. Contributions of bug fixes, tests,
documentation, examples, and focused features are welcome.

## Getting started

1. Fork the repository and create a branch from the default branch.
2. Install dependencies with `npm ci`.
3. Run the demo with `npm run dev` while developing.
4. Run `npm run build` before opening a pull request.

## Pull requests

- Keep each pull request focused on one change.
- Explain the user-facing behavior and include reproduction or verification
  steps.
- Add or update documentation and tests when behavior changes.
- Do not commit generated build output, local benchmark artifacts, or secrets.

## Reporting issues

Use the issue templates for bugs and feature requests. For a rendering or
interaction issue, include a minimal data sample, browser/version, and a
screenshot or recording when possible.

## Community expectations

Please follow the [Code of Conduct](CODE_OF_CONDUCT.md) in all project spaces.
